## デザインシステム準拠

### The Digital Curator - 主要原則

本テスト仕様は以下のデザイン原則に準拠しています:

1. **トーナルレイヤリング**: 影ではなく背景色の変化で奥行きを表現
   - テーブルヘッダー: #f1f4f6
   - 行ホバー: rgba(0, 32, 69, 0.04)

2. **ゴーストボーダー**: 透明度15%の境界線
   - rgba(85, 95, 113, 0.15)

3. **100% Blackの禁止**: テキストには#181c1eを使用（本仕様では#002045 primary）

4. **タイポグラフィコントラスト**:
   - Display/Headlines: Manrope（ページタイトル）
   - Body/UI: Inter（テーブル、フォーム）

5. **呼吸する余白**:
   - カードパディング: 16-24px
   - セクションギャップ: 16-32px

6. **角丸の一貫性**:
   - カード/パネル: 12px
   - フォーム要素: 6px

## テストケースの利用方法

### 開発者向け

```bash
# 全テストケースを確認
npm test -- tests/integration/frontend/page.test.tsx

# Activity History Pageのみ
npm test -- tests/integration/frontend/page.test.tsx -t "ActivityHistoryPage"

# 特定セクション（例: レイアウト）
npm test -- tests/integration/frontend/page.test.tsx -t "ActivityHistoryPage レイアウト"
```

### テストケースの実装手順

1. `test.todo()` を `test()` に変更
2. 必要なモック・データを準備
3. コンポーネントをレンダリング
4. アサーション（expect）を記述
5. テスト実行・デバッグ

### 例: test.todoをtestに変換

```typescript
// Before
test.todo("検索ボックスの背景色は#ffffff (white) である");

// After
test("検索ボックスの背景色は#ffffff (white) である", () => {
    vi.mocked(activitiesUseCase.getActivitiesFromLocal).mockResolvedValue([]);
    const { container } = render(
        <QueryClientProvider client={queryClient}>
            <MemoryRouter>
                <ActivityHistoryPage />
            </MemoryRouter>
        </QueryClientProvider>
    );
    
    const searchBox = container.querySelector('input[placeholder="活動を検索..."]');
    expect(searchBox).toHaveStyle({ backgroundColor: '#ffffff' });
});
```
